Project Plan — "ConnectKi Dating Website"

This document outlines the complete plan and the CSS layout system that will be used to build the dating website.

1. Website Topic & Purpose

Topic: A modern dating website where users browse profiles and submit their own.

Core purposes:

  • Dynamic profiles fetched from JSON.
  • Modal dialog with full user details.
  • Profile submission form.
  • LocalStorage implementation.

2. Page Structure (3 Required Pages)

PAGE 1 — Home (index.html)

  • Hero banner with title and description.
  • Featured profiles preview (3 cards).
  • Button linking to full profiles page.
  • SEO meta tags + favicon.

PAGE 2 — Profiles (profiles.html)

  • Dynamic grid of 15+ profiles.
  • Four data fields per profile.
  • Modal dialog with full information.
  • LocalStorage (theme preference or favorites).

PAGE 3 — About / Join Us (about.html)

  • Overview of how the site works.
  • HTML form with action to form-action.html.

3. Extra Pages (Not part of 3 required pages)

  • form-action.html: Displays submitted form data.
  • attributions.html: Lists external credits.

4. CSS Layout Plan (Very Important)

This describes the exact layout system used in your final website.

4.1 Color Palette

  • Main color: #c0392b (romantic red)
  • Dark Text: #2c3e50
  • Soft Background: #f8f8f8
  • Card Background: #ffffff
  • Button Hover Color: #e74c3c

4.2 Layout System

The website will use a combination of flexbox and CSS grid layouts:

  • Header navigation: Flexbox (horizontal for desktop, vertical for mobile).
  • Profile cards: CSS Grid (responsive).
  • Modal: Fixed overlay with centered content.
  • Form layout: Flex-column with spacing.
